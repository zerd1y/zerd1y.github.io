MySQL

SQL

  • SQL 是处理关系数据库的标准语言，用于插入、搜索、更新和删除数据库记录。
  • SQL 关键字不区分大小写：select 与 SELECT 相同

重要命令

  • SELECT - 从数据库中提取数据
  • UPDATE - 更新数据库中的数据
  • DELETE - 从数据库中删除数据
  • INSERT INTO - 将新数据插入数据库
  • CREATE DATABASE - 创建新数据库
  • ALTER DATABASE - 修改数据库
  • CREATE TABLE - 创建新表
  • ALTER TABLE - 修改表
  • DROP TABLE - 删除表
  • CREATE INDEX - 创建索引（搜索键）
  • DROP INDEX - 删除索引

SELECT

SELECT 语句用于从数据库中选择数据，返回的数据存储在结果表中，称为结果集。

SELECT DISTINCT 语句用于仅返回不同（不同）值。

WHERE

WHERE 子句用于筛选记录，它用于仅提取满足指定条件的记录。

  • 注意：WHERE 子句不仅用于 SELECT 语句，还用于 UPDATE， DELETE 等！

运算符：

ORDER BY

ORDER BY 关键字用于对结果集进行升序或降序排序。

认情况下，ORDER BY 关键字按升序对记录进行排序。要按降序对记录进行排序，请使用 DESC 关键字。

INSER INTO

INSERT INTO 语句用于在表中插入新记录

  • 为表的所有列添加值：（添加一行完整的数据）请确保值的顺序与表中的列的顺序相同

  • 指定要插入的列名称和值：添加一行中的部分数据

NULL Value

具有 NULL 值的字段是没有值的字段

如果表中的字段是可选的，则可以插入新记录或更新记录，而无需向此字段添加值。然后，该字段将使用 NULL 值保存

**注意：**NULL 值不同于零值或包含空格的字段。具有 NULL 值的字段是在创建记录期间留空的字段

测试 NULL 值

无法使用比较运算符（如 =、< 或 <>）测试 NULL 值

用 IS NULL 和 IS NOT NULL 运算符

  • IS NULL

  • IS NOT NULL

UPDATE

UPDATE 语句用于修改表中的现有记录

注意：更新表中的记录时要小心！请注意 UPDATE 语句中的 WHERE 子句。WHERE 子句指定应更新的记录。如果省略 WHERE 子句，表中的所有记录都将更新！

DELETE

DELETE 语句用于删除表中的现有记录。

注意：删除表中的记录时要小心！请注意 ``DELETE 语句。WHERE 子句指定应删除哪些记录。如果省略 WHERE 子句，表中的所有记录都将被删除！

LIMIT

LIMIT 子句用于指定要返回的记录数。

LIMIT 子句在具有数千条记录的大型表上非常有用。返回大量记录可能会影响性能。

MAX() 函数返回所选列的最大值

COUNT() AVG() SUM()

LIKE

LIKE 运算符用于 WHERE 子句中，用于在列中搜索指定的模式

通常有两个通配符与 LIKE 运算符结合使用：

  • %：百分号，表示零个、一个或多个字符
  • _：下划线符号，表示一个单个字符

百分号和下划线也可以组合使用

通配符用于替换字符串中的一个或多个字符。

通配符与 LIKE 一起使用 算子。LIKE 运算符用于 WHERE 子句中，用于在列中搜索指定的模式

  • %：替代零个、一个或多个字符
  • _：替代单个字符

IN

IN 运算符允许您在 WHERE 子句中指定多个值

IN 运算符是多个 OR 条件的简写

BETWEEN

BETWEEN 运算符选择给定范围内的值。值可以是数字、文本或日期。

BETWEEN 运算符是包含的：包括 begin 和 end 值。

Joins

JOIN 子句用于根据两个或多个表之间的相关列来组合它们中的行

MySQL 中支持的联接类型：

  • INNER JOIN：返回在两个表中具有匹配值的记录
  • LEFT JOIN：返回左表中的所有记录，以及右表中的匹配记录
  • RIGHT JOIN：返回右表中的所有记录，以及左表中的匹配记录
  • CROSS JOIN：返回两个表中的所有记录

INNER JOIN

INNER JOIN 关键字选择在两个表中具有匹配值的记录

LEFT JOIN

LEFT JOIN 关键字返回左表 （table1） 中的所有记录，以及右表 （table2） 中的匹配记录（如果有）

注意：LEFT JOIN 关键字返回左表 （Customers） 中的所有记录，即使右表 （Orders） 中没有匹配项。（显示NULL）

RIGHT JOIN

RIGHT JOIN 关键字返回右表 （table2） 中的所有记录，以及左表 （table1） 中的匹配记录（如果有）

注意：RIGHT JOIN 关键字返回右表 （Employees） 中的所有记录，即使左表 （Orders） 中没有匹配项。

CROSS JOIN

CROSS JOIN 关键字返回两个表（table1 和 table2）中的所有记录

注意：CROSS JOIN 可能会返回非常大的结果集

  • 结果集的行数将是两个表行数的乘积

注意：CROSS JOIN 关键字返回两个表中的所有匹配记录，无论另一个表是否匹配。因此，如果 “Customers” 中的某些行在 “Orders” 中没有匹配项，或者如果 “Orders” 中的行在 “Customers” 中没有匹配项，则也会列出这些行。

UNION

  • 每个带有 UNION 的 SELECT 语句必须具有相同数量的列
  • 每个 SELECT 语句中的列也必须按相同的顺序排列

GROUP BY

GROUP BY 语句将具有相同值的行分组到摘要行中，例如“查找每个国家/地区的客户数量”。

GROUP BY 语句通常与聚合函数 （COUNT() MAX() MIN() SUM() AVG()）按一个或多个列对结果集进行分组

HAVING

将 HAVING 子句添加到 SQL 中，因为 WHERE 关键字不能与聚合函数一起使用

HAVING 和 WHERE 的区别：

  1. 使用范围：
    • WHERE：用于过滤原始数据，在分组 (GROUP BY) 和聚合 (SUM, AVG, COUNT 等) 之前 进行筛选
    • HAVING：用于过滤分组后的数据，在 GROUP BY 之后 进行筛选，通常用于筛选聚合函数的结果
  2. 作用对象不同：
    • WHERE 作用于 单行数据，不能使用聚合函数
    • HAVING 作用于 分组后的数据，可以使用聚合函数
  3. 执行顺序：
    • 先执行 FROM，然后 WHERE 进行行过滤
    • 之后执行 GROUP BY 进行分组
    • 计算聚合函数后，HAVING 进行筛选

EXISTS

EXISTS 运算符用于测试子查询中是否存在任何记录

如果子查询返回一条或多条记录，则 EXISTS 运算符返回 TRUE

ANY ALL

ANY 和 ALL 运算符允许您在单个列值和一系列其他值之间执行比较

ANY 运算符：

  • 返回布尔值作为结果
  • 如果任何子查询值满足条件，则返回 TRUE

ANY 表示，如果对范围中的任何值执行作为 true，则条件将为 true

**注意：**运算符必须是标准比较运算符（=、<>、！=、>、>=、< 或 <=）

ALL 运算符：

  • 返回布尔值作为结果
  • 如果所有子查询值都满足条件，则返回 TRUE
  • 与 SELECT 、 WHERE、 HAVING 语句一起使用

ALL 表示仅当范围内的所有值都为 true 时，条件才为 true

**注意：**运算符必须是标准比较运算符（=、<>、！=、>、>=、< 或 <=）

INSERT INTO SELECT 语句从一个表中复制数据并将其插入到另一个表中

INSERT INTO SELECT 语句要求源表和目标表中的数据类型匹配

**注意：**目标表中的现有记录不受影响

CASE

CASE 语句遍历条件并在满足第一个条件时返回一个值（如 if-then-else 语句）。因此，一旦条件为 true，它将停止读取并返回结果。如果没有条件为 true，则返回 ELSE 子句中的值。

如果没有 ELSE 部分且没有条件为 true，则返回 NULL

NULL 函数

IFNULL()

IFNULL() 函数允许您在表达式为 NULL 时返回替代值

  • 如果 expr1 是 NULL，则返回 expr2，否则返回 expr1
  • 只能接受两个参数。

COALESCE()

  • 返回第一个非 NULL 的值，如果所有值都是 NULL，返回 NULL
  • 可以接受多个参数，按顺序检查哪个值不是 NULL，然后返回该值

注释

注释用于解释 SQL 语句的各个部分，或阻止执行 SQL 语句

单行注释：--

--和行尾之间的任何文本都将被忽略（不会执行）

多行注释：以 /*开头，以*/结尾

/* 和 */ 之间的任何文本都将被忽略
